## Commits

Format: `type(scope): message`

Types principaux:

- `feat`: fonctionnalité
- `fix`: correction
- `refactor`: amélioration interne
- `test`: validation/tests
- `docs`: documentation
- `chore`: maintenance/config

## PR Template (court)

1. Contexte
2. Changements
3. Pattern(s) concerné(s)
4. Résultat observé dans `Main`
5. Checklist de validation

## User Story / Issue template

- En tant que développeur
- Je veux implémenter `<pattern/fonctionnalité>`
- Afin de faire passer la section correspondante de `Main`
- Critères d'acceptation: sortie attendue + contraintes du TP
